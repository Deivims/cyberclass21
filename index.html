1. ¿Qué es un Dato?

Un dato es una representación simbólica (numérica, alfabética, alfanumérica, etc.) de un atributo o característica de una entidad. Por sí solo, un dato no tiene significado específico.

2. ¿Qué es información?

La información es un conjunto de datos organizados y procesados que tienen significado y son útiles para la toma de decisiones o para el cumplimiento de determinados objetivos.

¿Diferencia entre dato e información?

La diferencia principal entre dato e información radica en el significado y el contexto. Mientras que un dato es una representación simbólica de un atributo, la información es el resultado del procesamiento y la organización de datos que tienen un significado y utilidad específicos.

¿Tipos de datos?

Los tipos de datos pueden variar según el contexto, pero algunos tipos comunes incluyen datos numéricos (enteros, decimales), datos de texto, datos booleanos (verdadero/falso), datos de fecha y hora, entre otros.

¿Qué es un modelado de Datos?

El modelado de datos es el proceso de crear una representación estructurada y visual de los datos de un sistema o una organización, utilizando técnicas como diagramas de entidad-relación (ER) o modelos de datos.

Tipos de bases de Datos, según: - Variabilidad - Contenido - Modelo de base de datos

  • Variabilidad: Bases de datos estáticas (no cambian con el tiempo) y bases de datos dinámicas (se actualizan o modifican con el tiempo).
  • Contenido: Bases de datos de texto (orientadas a almacenar y recuperar texto), bases de datos multimedia (almacenan y gestionan contenido multimedia), bases de datos espaciales (para datos geográficos), entre otros.
  • Modelo de base de datos: Bases de datos relacionales (basadas en el modelo relacional), bases de datos NoSQL (modelos de datos no relacionales), bases de datos orientadas a objetos, entre otros.

¿Qué es el modelo relacional?

El modelo relacional es un modelo de datos que organiza los datos en tablas (relaciones), donde cada fila representa una entidad y cada columna representa un atributo. Las relaciones entre las tablas se establecen a través de claves primarias y claves foráneas.

¿Describa los elementos del Diagrama entidad Relacional?

Los elementos del Diagrama Entidad-Relación incluyen entidades (objetos sobre los que se recopilan datos), atributos (características o propiedades de las entidades) y relaciones (asociaciones entre entidades).

¿Qué es una entidad?

Una entidad es un objeto o concepto del mundo real que puede ser identificado y del cual se pueden recopilar datos. En un modelo de datos, una entidad se representa como una tabla en la que cada fila representa una instancia de la entidad.

¿Qué es un atributo y sus tipos?

Un atributo es una característica o propiedad de una entidad que describe algún aspecto de la misma. Los tipos de atributos pueden incluir atributos simples (valores individuales), atributos compuestos (divididos en partes más pequeñas) y atributos derivados (calculados a partir de otros atributos).

¿Qué es una relación?

Una relación es una asociación entre dos o más entidades que se establece en base a reglas de negocio o requisitos del sistema. En un modelo relacional, las relaciones se representan mediante claves foráneas que vinculan las tablas correspondientes.

¿Qué son las llaves primarias, foránea y sus características?

La llave primaria es un atributo (o conjunto de atributos) que identifica de forma única a cada fila en una tabla. La llave foránea es un atributo (o conjunto de atributos) que establece una relación entre dos tablas, haciendo referencia a la llave primaria de otra tabla. Las llaves primarias deben ser únicas y no nulas, mientras que las llaves foráneas deben hacer referencia a una llave primaria existente.

¿Qué son las reglas del Negocio?

Las reglas de negocio son declaraciones que definen o regulan algún aspecto del negocio, como políticas, procedimientos o restricciones. En el contexto de la modelización de datos, las reglas de negocio pueden influir en el diseño y la estructura de la base de datos.

¿Qué es la cardinalidad o mapeo?

La cardinalidad o mapeo describe la relación entre entidades en un modelo de datos y especifica el número máximo de instancias de una entidad que pueden estar relacionadas con una instancia de otra entidad. Por ejemplo, una relación uno a uno, uno a muchos o muchos a muchos.
